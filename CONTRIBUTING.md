## üõ†Ô∏è How to Contribute

When you're ready to submit code, please follow this standard workflow:

1. **Fork** the repository to your own GitHub account.
2. **Clone** the forked repository to your local machine.
3. **Create a new branch** for your contribution: 
   `git checkout -b feature/my-feature`
4. **Make your changes** and ensure they are well-tested.
5. **Stage and commit** your changes with a descriptive message: 
   `git add .`
   `git commit -m "Add new feature: brief description here"`
6. **Push** your changes to your branch: 
   `git push origin feature/my-feature`
7. **Create a Pull Request** against our main repository, providing a clear and detailed description of your changes.

üí° Tip: Don't worry about maintaining a perfectly clean commit history on your branch or force-pushing to tidy things up. We squash merge every PR, so feel free to just keep committing and pushing as you address feedback!

## üìù Code Style
To keep the codebase clean and maintainable, please ensure your PRs align with these principles:
* **Follow existing conventions:** Match the code style of the surrounding files.
* **Be clear and concise:** Write code that is easy to read, with meaningful comments where necessary.
* **Name things well:** Use descriptive variable and function names.
